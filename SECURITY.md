# Security Policy

## Reporting a Vulnerability

We take security seriously at Unbound Labs. If you discover a security vulnerability, please report it responsibly.

### How to Report

1. **Do not** create a public GitHub issue
2. Email us at: **support@unboundlabs.dev**
3. Include "SECURITY" in the subject line

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fixes (if any)

### Response Timeline

- **Initial response:** Within 48 hours
- **Status update:** Within 7 days
- **Resolution target:** Within 30 days (depending on severity)

### What to Expect

1. Acknowledgment of your report
2. Assessment of the vulnerability
3. Regular updates on progress
4. Credit in our security acknowledgments (if desired)

### Scope

This policy applies to:
- The Unbound Labs website
- All repositories under the Unbound Labs organization
- Related infrastructure and services

### Out of Scope

- Social engineering attacks
- Physical attacks
- Denial of service attacks
- Issues in third-party dependencies (report to upstream)

## Security Best Practices

When contributing:
- Never commit secrets or credentials
- Use environment variables for sensitive data
- Keep dependencies updated
- Follow secure coding practices

## Acknowledgments

We appreciate researchers who help keep our projects secure. Contributors will be acknowledged (with permission) in our security hall of fame.

Thank you for helping keep Unbound Labs safe.
